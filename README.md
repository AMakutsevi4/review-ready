# review-ready

Замечания к проекту:

1. В application.properties при настройках взаимодействия с БД лучше использовать validate;
2. В сущности нежелательно использовать @Data;
3. Контроллер работает напрямую с сущностью;
4. Отсутствуют тесты;
5. В сервисе в методе calculateCommission используются константные значения;
6. В методе calculateCommission избыточная логика считает и сохраняет комиссию;
7. Нет единого стиля написания когда. Гже-то внедряются зависимости через @RequiredArgsConstructor, где-то через конструктор;
8. Лишние пробелы в каждом классе;
9. В репозитории указана некорректная аннотация.
10. Коммит сделан в ветку main.
